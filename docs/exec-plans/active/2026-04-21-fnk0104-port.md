# Plan: Port clawd-tank firmware to Freenove ESP32-S3 2.8" (fnk0104)

**Goal:** Run clawd-tank on the Freenove ESP32-S3 2.8" (fnk0104), coexisting with the existing Waveshare ESP32-C6-LCD-1.47 build.
**Status:** Active
**Started:** 2026-04-21

## Context

Hardware today is only the Waveshare ESP32-C6-LCD-1.47 (ESP32-C6, 320×172 ST7789, no PSRAM, 8 MB flash). The new target is the Freenove fnk0104 (ESP32-S3, 320×240 ILI9341, 8 MB OPI PSRAM, 16 MB QIO-OPI flash, FT6336G capacitive touch, WS2812B on a different pin, no hardware user button).

The user's pixel-agents-esp32 project already runs on fnk0104 and is the hardware-wiring reference. That project is Arduino + TFT_eSPI; clawd-tank is ESP-IDF native + LVGL 9.5 + `esp_lcd`, so pin maps and orientation flags carry over but all init code must be translated from TFT_eSPI macros to the `esp_lcd` panel-driver API.

Both boards must build from the same source tree going forward (Kconfig board selector + per-board `#ifdef` guards + `sdkconfig.defaults.<target>`). The C6 build is the canonical regression target.

## Steps

### Phase 1 — Toolchain & sdkconfig
- [ ] Move current C6 settings to `sdkconfig.defaults.esp32c6`; keep board-agnostic settings in `sdkconfig.defaults`.
- [ ] Create `sdkconfig.defaults.esp32s3` with PSRAM (OPI 80 MHz), QIO-OPI flash, 16 MB flash size, dual-core FreeRTOS.
- [ ] Extend `firmware/main/Kconfig.projbuild` with `choice CLAWD_BOARD` (`BOARD_WAVESHARE_C6_LCD147` default, `BOARD_FREENOVE_S3_28`).
- [ ] Verify `idf.py set-target esp32s3 && idf.py reconfigure` succeeds.

### Phase 2 — board_config.h extraction (no behavior change)
- [ ] Create `firmware/main/board_config.h` with `#ifdef CONFIG_BOARD_*` blocks for all pin/resolution constants.
- [ ] Edit `display.c`, `button.c`, `rgb_led.c` to `#include "board_config.h"` and remove local `#define`s.
- [ ] Rebuild for C6, confirm identical behavior.

### Phase 3 — ILI9341 panel driver
- [ ] Add `espressif/esp_lcd_ili9341` to `idf_component.yml`.
- [ ] In `display.c`, `#ifdef`-gate the panel-creation call (ST7789 for C6, ILI9341 for fnk0104).
- [ ] Skip `esp_lcd_panel_set_gap()` when gap = 0 (fnk0104 has no y_gap).
- [ ] Document orientation/RGB-order flags as bring-up TODOs (final values confirmed on hardware).

### Phase 4 — Scene/UI layout generalization
- [ ] Replace hardcoded `172` in `scene.c`, `notification_ui.c` with `SCENE_HEIGHT` / `SCREEN_H` sourced from `board_config.h`.
- [ ] Derive `FEATURED_H_EXPANDED` from `SCREEN_H` (already does); verify no other hardcoded heights break.
- [ ] Star placement: keep existing coords (they use y < 30 px, safe in either display).
- [ ] Confirm simulator still renders correctly (simulator uses SDL and builds its own scene.c copy — changes must not regress it).

### Phase 5 — Peripheral pins & button compile-out
- [ ] RGB LED pin 8 → 42 on fnk0104 (one line in `board_config.h`).
- [ ] `button.c` compile-out and `main.c` skip `button_init()` when `CONFIG_BOARD_FREENOVE_S3_28=y`.

### Phase 6 — FT6336G capacitive touch
- [ ] Add `espressif/esp_lcd_touch_ft5x06` managed component.
- [ ] New `firmware/main/touch.c/h` mirroring `button.c` — I2C init, INT ISR, debounce, post `BLE_EVT_NOTIF_CLEAR` on tap.
- [ ] Coordinate transform for landscape copied from `pixel-agents-esp32/firmware/src/touch_input.cpp`.
- [ ] Wire into `main.c` init on fnk0104 only.

### Phase 7 — Battery ADC + HUD
- [ ] New `firmware/main/battery.c/h` — ADC oneshot on GPIO 9, 2× divider, `adc_cali_curve_fitting` calibration, EMA smoothing (α=0.15), 5 s timer.
- [ ] New `scene_set_battery(uint8_t pct, bool charging)` entry point; small battery glyph top-right of scene.
- [ ] Simulator stub returns 75 %.

### Phase 8 — Memory strategy
- [ ] `MAX_SLOTS = 8` on fnk0104 (S3 has PSRAM + more SRAM).
- [ ] Sprite frame buffers allocated via `heap_caps_malloc(..., MALLOC_CAP_SPIRAM)` on fnk0104 only; LVGL flush buffers stay in internal DMA SRAM.

### Phase 9 — Verify & document
- [ ] Build both targets (`idf.py set-target esp32c6 && idf.py build`, same with `esp32s3`).
- [ ] Run `firmware/test && make test` (should still be host-only, target-agnostic).
- [ ] Build simulator (`cd simulator && cmake --build build`).
- [ ] Run host tests (`cd host && .venv/bin/pytest -v`).
- [ ] Update `ARCHITECTURE.md`, `AGENTS.md` (build commands section), `docs/QUALITY_SCORE.md`, `docs/exec-plans/tech-debt-tracker.md` (log any deferred items), `README.md`.
- [ ] Move this plan to `docs/exec-plans/completed/` and append a release-log entry in `docs/PLANS.md`.
- [ ] **Hardware bring-up (user-side):** flash fnk0104, tune ILI9341 orientation flags and RGB/BGR order by eye, verify touch dismisses notifications, verify battery HUD.

## Decisions

- **2026-04-21:** Coexist via Kconfig. Both boards build from the same source tree; the C6 build is the regression guarantor.
- **2026-04-21:** Include FT6336G touch and battery HUD in this port (not deferred). Full fnk0104 feature parity shipped at once.
- **2026-04-21:** Keep the 8 MB partition layout on the 16 MB-flash fnk0104 for now. Wasted flash is cheap; partition-table rewrites invite OTA questions this port doesn't need to answer.

## Open questions

- Orientation flags for ILI9341 (`swap_xy`, `mirror_x`, `mirror_y`) and RGB vs BGR element order can't be confirmed without flashing. Initial values are best-guess; final tuning happens during hardware bring-up.
