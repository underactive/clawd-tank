# Plan: Port clawd-tank firmware to Freenove ESP32-S3 2.8" (fnk0104)

**Goal:** Run clawd-tank on the Freenove ESP32-S3 2.8" (fnk0104), coexisting with the existing Waveshare ESP32-C6-LCD-1.47 build.
**Status:** Completed
**Started:** 2026-04-21
**Completed:** 2026-04-21

## Context

Hardware today is only the Waveshare ESP32-C6-LCD-1.47 (ESP32-C6, 320×172 ST7789, no PSRAM, 8 MB flash). The new target is the Freenove fnk0104 (ESP32-S3, 320×240 ILI9341, 8 MB OPI PSRAM, 16 MB QIO-OPI flash, FT6336G capacitive touch, WS2812B on a different pin, no hardware user button).

The user's pixel-agents-esp32 project already runs on fnk0104 and is the hardware-wiring reference. That project is Arduino + TFT_eSPI; clawd-tank is ESP-IDF native + LVGL 9.5 + `esp_lcd`, so pin maps and orientation flags carry over but all init code must be translated from TFT_eSPI macros to the `esp_lcd` panel-driver API.

Both boards must build from the same source tree going forward (Kconfig board selector + per-board `#ifdef` guards + `sdkconfig.defaults.<target>`). The C6 build is the canonical regression target.

## Steps

### Phase 1 — Toolchain & sdkconfig
- [x] Move current C6 settings to `sdkconfig.defaults.esp32c6`; keep board-agnostic settings in `sdkconfig.defaults`.
- [x] Create `sdkconfig.defaults.esp32s3` with PSRAM (OPI 80 MHz), QIO-OPI flash, 16 MB flash size, dual-core FreeRTOS.
- [x] Extend `firmware/main/Kconfig.projbuild` with `choice CLAWD_BOARD` (`BOARD_WAVESHARE_C6_LCD147` default, `BOARD_FREENOVE_S3_28`).
- [x] Verify `idf.py set-target esp32s3 && idf.py reconfigure` succeeds.

### Phase 2 — board_config.h extraction (no behavior change)
- [x] Create `firmware/main/board_config.h` with `#ifdef CONFIG_BOARD_*` blocks for all pin/resolution constants.
- [x] Edit `display.c`, `button.c`, `rgb_led.c` to `#include "board_config.h"` and remove local `#define`s.
- [x] Rebuild for C6, confirm identical behavior.

### Phase 3 — ILI9341 panel driver
- [x] Add `espressif/esp_lcd_ili9341` to `idf_component.yml`.
- [x] In `display.c`, `#ifdef`-gate the panel-creation call (ST7789 for C6, ILI9341 for fnk0104).
- [x] Skip `esp_lcd_panel_set_gap()` when gap = 0 (fnk0104 has no y_gap).
- [x] Document orientation/RGB-order flags as bring-up TODOs (final values confirmed on hardware).

### Phase 4 — Scene/UI layout generalization
- [x] Replace hardcoded `172` in `scene.c`, `notification_ui.c` with `SCENE_HEIGHT` / `SCREEN_H` sourced from `board_config.h`.
- [x] Derive `FEATURED_H_EXPANDED` from `SCREEN_H` (already does); verify no other hardcoded heights break.
- [x] Star placement: keep existing coords (they use y < 30 px, safe in either display).
- [x] Confirm simulator still renders correctly (simulator uses SDL and builds its own scene.c copy — changes must not regress it).

### Phase 5 — Peripheral pins & button compile-out
- [x] RGB LED pin 8 → 42 on fnk0104 (one line in `board_config.h`).
- [x] `button.c` compile-out and `main.c` skip `button_init()` when `CONFIG_BOARD_FREENOVE_S3_28=y`.

### Phase 6 — FT6336G capacitive touch
- [x] Add `espressif/esp_lcd_touch_ft5x06` managed component.
- [x] New `firmware/main/touch.c/h` mirroring `button.c` — I2C init, INT ISR, debounce, post `BLE_EVT_NOTIF_CLEAR` on tap.
- [x] Coordinate transform for landscape copied from `pixel-agents-esp32/firmware/src/touch_input.cpp`.
- [x] Wire into `main.c` init on fnk0104 only.

### Phase 7 — Battery ADC + HUD
- [x] New `firmware/main/battery.c/h` — ADC oneshot on GPIO 9, 2× divider, `adc_cali_curve_fitting` calibration, EMA smoothing (α=0.15), 5 s timer.
- [x] New `scene_set_battery(uint8_t pct, bool charging)` entry point; small battery glyph top-right of scene.
- [x] Simulator stub returns 75 %.

### Phase 8 — Memory strategy
- [x] `MAX_SLOTS = 8` on fnk0104 (S3 has PSRAM + more SRAM).
- [x] Sprite frame buffers allocated via `heap_caps_malloc(..., MALLOC_CAP_SPIRAM)` on fnk0104 only; LVGL flush buffers stay in internal DMA SRAM.

### Phase 9 — Verify & document
- [x] Build both targets (`idf.py set-target esp32c6 && idf.py build`, same with `esp32s3`).
- [x] Run `firmware/test && make test` (should still be host-only, target-agnostic).

- [x] Run host tests (`cd host && .venv/bin/pytest -v`).
- [x] Update `ARCHITECTURE.md`, `AGENTS.md` (build commands section), `docs/QUALITY_SCORE.md`, `docs/exec-plans/tech-debt-tracker.md` (log any deferred items), `README.md`.

- [x] **Hardware bring-up (user-side):** flash fnk0104, tune ILI9341 orientation flags and RGB/BGR order by eye, verify touch dismisses notifications, verify battery HUD.

## Decisions

- **2026-04-21:** Coexist via Kconfig. Both boards build from the same source tree; the C6 build is the regression guarantor.
- **2026-04-21:** Include FT6336G touch and battery HUD in this port (not deferred). Full fnk0104 feature parity shipped at once.
- **2026-04-21:** Keep the 8 MB partition layout on the 16 MB-flash fnk0104 for now. Wasted flash is cheap; partition-table rewrites invite OTA questions this port doesn't need to answer.

## Completion notes

Phases 1-8 shipped in `cb22583`. Phase 9 verification surfaced two
Phase-9-worthy regressions that the ASan-hang-before-main pre-existing
toolchain bug had been masking:

- `d6f1c44` — `BOARD_RGB_LED_GPIO` missing from the SIMULATOR block of
  `board_config.h`, breaking the static simulator build that the `.app`
  bundle depends on.
- `c53e240` — `test_config_store.c` still asserted the pre-port
  brightness default (102) instead of the new 230. Rewrote the
  assertion to source from `CONFIG_DEFAULT_*` macros so future default
  changes don't re-stale this test.
- `4b24303` — Dropped `-fsanitize=address` from `firmware/test/Makefile`
  to work around macOS 26 `AsanInitFromRtl` init-recursion deadlock;
  UBSan retained. Logged in `tech-debt-tracker.md`.
- `6107c88` — Pinned `setuptools` + `py2app` in `requirements-dev.txt`
  so a fresh host venv can build the `.app` without extra manual pip
  installs on Python ≥ 3.12.

All 23 firmware C tests pass under UBSan; 222 host pytest tests pass.
C6 and S3 firmware targets both build clean.
